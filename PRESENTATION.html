🏗️ Unified Architecture

How all pieces fit together. Each layer has a clear source and responsibility.

🌐 API Layer

← Our server/src/

Hono HTTP server with full perimeter security

  • Request ID, CORS, security headers, payload limits
  • Rate limiting, auth backstop, OTEL metrics middleware
  • Versioned REST API (/api/v1) with Zod validation
  • SSE bus for real-time events
  • Health check, metrics endpoint
  • API 404 guard (JSON never SPA)

🔐 Auth & Security

← Our server/src/ + Goose patterns

Full auth stack with audit trail

  • API key / scope-based authentication
  • Kill switch (emergency stop all mutations)
  • Audit chain — every mutation recorded in DB
  • PII redaction, prompt injection detection
  • Jailbreak detection (from OmniRoute guardrails)
  • Sandbox for untrusted code execution

🤖 AI Gateway

← Portkey (integrated) + Goose (Rust native)

Unified interface to 150+ AI providers

  • Portkey: OpenAI, Anthropic, Azure, Bedrock, Google, 130+ more
  • Goose: Rust-native OpenAI, Anthropic, Ollama clients
  • Streaming, tool calls, vision, audio, embeddings
  • Caching (memory, Redis, file, Cloudflare KV)
  • Rate limiting per provider, per key
  • Automatic retry with exponential backoff

🔀 LLM Routing

← OmniRoute (integrated)

Smart routing between models and providers

  • Model assessment & categorization
  • Cost-based routing (cheapest capable model)
  • Fallback chains on failure/degradation
  • Tag routing (route by content type)
  • Lockout policy for failing providers
  • Quota management and provider expiration

🧠 Agent Engine

← Our services/ + Goose patterns + Gemini A2A

Multi-agent orchestration

  • Agent lifecycle management (create, run, pause, stop)
  • DAG-based agent pipelines (from our graph-engine)
  • Agent-to-Agent communication (from Gemini A2A)
  • P2P swarm mode for distributed agents
  • Task worker queue for background jobs
  • Self-improvement harness (agents that improve themselves)

  • Automatic MCP tool discovery and caching

  • Auto-generated embeddings for semantic search
  • Audit log for every data change
  • Brain export/import/compression
  • Obsidian vault sync (bidirectional)

Surgical Adaptation Process

Every file we copy gets minimal changes — only what's needed to fit our project structure. We never rewrite working code.

1️⃣ Copy source files

Copy *.rs files from goose_repo/crates/ into our crates/ directory.

  • File-by-file copy, preserving all logic
  • Rename crate from goose-providers → agentic-os-providers
  • Keep same module structure

2️⃣ Rewrite Cargo.toml

Create new Cargo.toml using our workspace deps instead of Goose's.

  • Replace { workspace = true } references to our workspace
  • Drop optional deps (goose-local-inference, AWS providers)
  • Remove vendor paths (tree-sitter, v8)
  • Point rmcp to our version

3️⃣ Adapt the lib.rs

Rewrite lib.rs to export modules under our crate naming.

  • Change pub use goose_provider_types::...
    → pub use agentic_os_core::...
  • Keep internal module structure identical
  • Preserve all re-exports

4️⃣ Fix import paths

Update internal crate imports to use our crate names.

  • goose-provider-types → agentic-os-provider-types
  • goose-providers → agentic-os-providers
  • Only import paths change — zero logic changes

5️⃣ Wire into our modules

Create adapter modules that connect Goose providers to our API.

  • Our config system → Goose ProviderConfig
  • Our error types → Goose ProviderError
  • Our Session → Goose Conversation
  • Minimal glue code

6️⃣ Run cargo check

Fix any compilation issues. Since we preserve logic, there should be zero behaviour changes — only path/name changes.

  • cargo check -p agentic-os-provider-types
  • cargo check -p agentic-os-providers
  • Fix any missing features

#### 3.3.1 Production fix landed with this baseline: DLP scanner OOM

`tests/dlp-scanner.test.ts` (and therefore ANY flagged payload in production)
crashed the worker via an infinite loop → heap exhaustion. Root cause:
`scan()` iterates matches with `RegExp.exec()`, which only advances
`lastIndex` for `/g` patterns; the pattern table lacked the `/g` flag. Fixed
in `server/src/services/dlp-scanner.ts` (patterns now global; redaction
rewritten as a single pass over original-text spans, which also removes a
duplicate-secret corruption bug). The suite can now run to completion for the
first time in this repository.

production migrations `0049/0050/0051` and the full R1 repository, idempotency,
append-only trigger, governed-capability, policy-API, and restart-durability

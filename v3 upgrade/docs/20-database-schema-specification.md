# NEXUS V3 — Complete Database Schema Specification
## PostgreSQL 16 + Drizzle ORM + pgvector
## Every Table, Every Column, Every Constraint

> **Purpose:** This document is the authoritative source for all database schema decisions. A low-level AI can create migrations, write queries, and validate data integrity using this spec alone. Every table includes: Drizzle definition, SQL equivalent, column constraints, indexes, relationships, and sample queries.

---

## CONVENTIONS

| Convention | Rule |
|---|---|
| Naming | `snake_case` for DB tables/columns, `camelCase` for Drizzle TS names |
| Primary keys | `id` — `uuid` (generated by `genRandomUUID()`) for most tables, `bigserial` for high-volume tables (audit_log) |
| Timestamps | `created_at` and `updated_at` on every mutable table (Drizzle `timestamp().defaultNow().notNull()`) |
| Soft delete | `deleted_at: timestamp()` on tables that support soft-delete |
| Text fields | `text` for unbounded, `varchar(N)` for bounded strings |
| Foreign keys | Always indexed. Use `onDelete: 'cascade'` for ownership, `onDelete: 'set null'` for optional references |
| Vector columns | `vector(1536)` for OpenAI embeddings, indexed with HNSW (`ivfflat` as fallback) |
| JSON columns | Use `jsonb` — supports indexing, partial updates, and path queries |
| Enums | Drizzle `pgEnum` for small fixed sets |

## TABLE 1: `memories` — Episodic & Semantic Memory Store
**Source file:** `server/src/db/schema.ts`
**Phase:** 4 (Memory System)

```typescript
// Drizzle definition (from server/src/db/schema.ts)
export const memories = pgTable('memories', {
  id: uuid('id').defaultRandom().primaryKey(),
  type: varchar('type', { length: 50 }).notNull().default('episodic'), // 'episodic' | 'semantic' | 'working' | 'procedural'
  projectId: uuid('project_id').references(() => projects.id, { onDelete: 'cascade' }),
  agentId: uuid('agent_id'),
  content: text('content').notNull(),
  embedding: vector('embedding', { dimensions: 1536 }),
  metadata: jsonb('metadata').default({}).notNull(),
  importance: integer('importance').default(0).notNull(), // 0-10 scale
  accessCount: integer('access_count').default(0).notNull(),
  lastAccessedAt: timestamp('last_accessed_at'),
  expiresAt: timestamp('expires_at'), // null = never expires
  tags: text('tags').array(),
  source: varchar('source', { length: 100 }), // 'user_input' | 'agent_thought' | 'tool_result' | 'external'
  checksum: varchar('checksum', { length: 64 }), // SHA-256 for deduplication
  createdAt: timestamp('created_at').defaultNow().notNull(),
  updatedAt: timestamp('updated_at').defaultNow().notNull(),
  deletedAt: timestamp('deleted_at'),
});
```

**SQL Equivalent:**
```sql
CREATE TABLE memories (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  type VARCHAR(50) NOT NULL DEFAULT 'episodic',
  project_id UUID REFERENCES projects(id) ON DELETE CASCADE,
  agent_id UUID,
  content TEXT NOT NULL,
  embedding VECTOR(1536),
  metadata JSONB NOT NULL DEFAULT '{}',
  importance INTEGER NOT NULL DEFAULT 0 CHECK (importance >= 0 AND importance <= 10),
  access_count INTEGER NOT NULL DEFAULT 0,
  last_accessed_at TIMESTAMP WITH TIME ZONE,
  expires_at TIMESTAMP WITH TIME ZONE,
  tags TEXT[],
  source VARCHAR(100),
  checksum VARCHAR(64),
  created_at TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT NOW(),
  updated_at TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT NOW(),
  deleted_at TIMESTAMP WITH TIME ZONE
);

-- Indexes
CREATE INDEX idx_memories_type ON memories(type);
CREATE INDEX idx_memories_project_id ON memories(project_id);
CREATE INDEX idx_memories_agent_id ON memories(agent_id);
CREATE INDEX idx_memories_importance ON memories(importance DESC);
CREATE INDEX idx_memories_created_at ON memories(created_at DESC);
CREATE INDEX idx_memories_checksum ON memories(checksum) WHERE checksum IS NOT NULL;
CREATE INDEX idx_memories_tags ON memories USING GIN(tags);
CREATE INDEX idx_memories_metadata ON memories USING GIN(metadata);
-- Vector index (HNSW — requires pgvector 0.7+)
CREATE INDEX idx_memories_embedding ON memories USING hnsw (embedding vector_cosine_ops) WITH (m = 16, ef_construction = 200);
```

**Sample Queries:**
```sql
-- Semantic search: find similar memories
SELECT id, content, 1 - (embedding <=> $query_embedding) AS similarity
FROM memories
WHERE type = 'semantic' AND deleted_at IS NULL
ORDER BY embedding <=> $query_embedding
LIMIT 10;

-- Agent's recent episodic memory
SELECT content, created_at, metadata
FROM memories
WHERE agent_id = $agent_id AND type = 'episodic' AND deleted_at IS NULL
ORDER BY created_at DESC
LIMIT 50;

-- Memory consolidation candidates (old, low-importance)
SELECT id, content, importance, created_at
FROM memories
WHERE created_at < NOW() - INTERVAL '7 days' AND importance < 3 AND deleted_at IS NULL
ORDER BY importance ASC, created_at ASC
LIMIT 100;
```

## TABLE 5: `audit_log` — Append-Only Audit Trail (Hash-Chained)

```typescript
export const auditLog = pgTable('audit_log', {
  id: bigserial('id', { mode: 'bigint' }).primaryKey(), // Bigserial for fast sequential writes
  eventType: varchar('event_type', { length: 100 }).notNull(),
  actorType: varchar('actor_type', { length: 50 }).notNull(), // 'user' | 'agent' | 'system' | 'api_key'
  actorId: varchar('actor_id', { length: 255 }).notNull(),
  resourceType: varchar('resource_type', { length: 100 }),
  resourceId: varchar('resource_id', { length: 255 }),
  action: varchar('action', { length: 50 }).notNull(), // 'create' | 'read' | 'update' | 'delete' | 'execute' | 'login' | 'logout'
  payload: jsonb('payload').default({}).notNull(),
  ipAddress: varchar('ip_address', { length: 45 }),
  userAgent: text('user_agent'),
  previousHash: varchar('previous_hash', { length: 64 }).notNull(), // SHA-256 of previous row
  hash: varchar('hash', { length: 64 }).notNull(), // SHA-256 of this row's content
  partitionDate: timestamp('partition_date').notNull().defaultNow(), // For partition pruning
  createdAt: timestamp('created_at').defaultNow().notNull(),
});

// Partition by month for performance
// Audit tables are partitioned — see 0002_audit_partitions.sql
```

**Tiered Storage (Phase 10):**
| Age | Storage | Retention |
|---|---|---|
| < 30 days | `audit_log` (hot partition) | Fast query |
| 30-365 days | `audit_log_archive` (warm) | Compressed, queryable |
| > 1 year | S3/Glacier (cold) | Compliance only |

## DATA RETENTION POLICY (Phase 10 implementation)

| Table | Hot (fast) | Warm | Cold (archive/delete) |
|---|---|---|---|
| memories | All active | Importance < 3 after 30d | After 90d → compress |
| audit_log | 30 days | 31-365 days partitioned | > 365d → S3/Glacier |
| trajectory_logs | 7 days | 8-90 days | After 90d → delete |
| token_ledger | 90 days | 91-365 days | > 365d → aggregate only |
| sandbox_executions | 30 days | 31-90 days | After 90d → delete |
| state_snapshots | 14 days | — | After 14d → delete |
| tool_receipts | 30 days | 31-90 days | After 90d → delete |

## VALIDATION RULES (Enforced in application layer, not DB)

| Rule | Table | Logic |
|---|---|---|
| Memory importance 0-10 | memories | Application validates before insert |
| Agent temperature 0-2 | agents | Application clamps to [0, 2] |
| Cron expression valid | cron_jobs | Application validates with cron-parser |
